# Origin & provenance

Agentry is a **clean-room reimplementation** of an agentic engine that has run in production inside a
multi-tenant SaaS — powering an AI agent with **72 tools** (autonomous content/site generation, live
progress streaming, crash-resume). That original is **proprietary**: it's tightly coupled to a private
platform's service SDKs, multi-tenancy, quota, and persistence, so it can't be open-sourced verbatim
(it wouldn't even compile standalone). Agentry extracts the reusable core, strips all that coupling,
and generalizes it.

This document shows **sanitized, abridged excerpts** of the original — *framework plumbing only*
(no business logic, no domain tools, no service integrations) — so the lineage is concrete, alongside
how Agentry improves on each piece.

## The original tool contract (sanitized)

```csharp
public interface ITool
{
    string Name { get; }
    string DisplayLabel { get; }
    Task<ToolResult> ExecuteAsync(ToolCall toolCall, ToolSessionContext sessionContext, CancellationToken ct);
}
```

Description + schema were attached via a static `Definition`, generated by reflection (NJsonSchema)
from a nested input type plus an embedded markdown description:

```csharp
public static readonly ToolDefinition Definition = new(
    Name: ToolName,
    Description: typeof(MyTool).LoadDescription(),   // embedded .md resource
    InputSchema: typeof(MyInput).GenerateSchema());  // NJsonSchema from the C# type
```

**Agentry improves this:** `Description` and `InputSchema` move *onto the interface* (no unenforced
static convention); the session type becomes a generic `TContext` (so the contract is reusable, not
welded to one domain); and a typed `Tool<TInput,TContext>` base deserializes + validates arguments for
you. Schema generation is dependency-free (no NJsonSchema).

## The original agentic loop (sanitized, abridged)

```csharp
const int maxIterations = 50;
var iteration = startIteration;
while (iteration++ < maxIterations)
{
    if (ct.IsCancellationRequested) { /* mark Interrupted, persist, throw */ }

    var response = await CallModelWithToolsAsync(model, apiMessages, systemPrompt, ct);
    if (!response.Success) { /* retry transient 429/503/529 with backoff, else fail */ }

    // persist assistant turn; accumulate tokens + cost
    if (response.StopReason == StopReasons.EndTurn) break;          // done
    if (response.StopReason == StopReasons.ToolUse)
    {
        foreach (var call in response.ToolCalls)
        {
            var result = await toolExecutor.ExecuteAsync(call, sessionContext, ct);
            // persist tool_use + tool_result; append tool_result block
        }
        continue;   // append one user turn carrying all tool results, loop again
    }
    if (response.StopReason == StopReasons.MaxTokens) break;        // truncated

    // save progress; brief throttle
}
```

**Agentry improves this:** the loop is provider-agnostic (it talks to `IChatModel`, not the Anthropic
wire format); persistence is inverted behind `IConversationStore` (the original wrote EF rows *inline*
in the loop); progress is surfaced as a neutral `AgentEvent` stream instead of being implied by DB
writes; and per-run state is a generic `TContext` instead of a domain-specific session object.

## What was deliberately left behind

The 72 concrete tools, their service-SDK calls, multi-tenancy, quota enforcement, symbolic-ID
resolution, prompt content, and all business/domain logic stay in the private platform. **Agentry is
the engine only** — the reusable machinery, none of the product.
